# Universal Workflow


This document describes how information moves through ShadowVault.

The objective is to identify, develop, and connect valuable ideas.

README:[README](../../README.md)

---


## The Path of an Idea

Every idea follows a similar path: 

```text
External World  
│  
▼  
Capture (00 - Inbox / Daily Note)  
│  
▼  
Source (01 - Sources) – raw material  
│  
▼  
Literature Note (02 - Literature Notes) – your summary & reaction  
│  
▼  
Permanent Note (03 - Permanent Notes) – atomic, evergreen idea  
│  
▼  
MOC / Project (04 - MOCs / 05 - Projects) – curation & action
```

Not every note completes this journey. Some fleeting thoughts are discarded. Some [SOURCES](SOURCES) are consumed and summarised without becoming permanent notes. The goal is **not to process everything** – it is to let important ideas rise.

## Note Maturity Model
Notes carry a `growth` frontmatter field. This replaces vague "status" with a clear development stage.

| Stage | Emoji | Meaning |
|-------|-------|---------|
| `seedling` | 🌱 | Raw capture. Rough or empty. Needs development. |
| `fern` | 🌿 | Partially developed. Has some explanation. Needs more thinking. |
| `incubator` | 🔆 | Complete thought. Can stand alone. Not yet linked. |
| `evergreen` | 🌲 | Polished, atomic, linked, and connected. |

Update the `growth` field manually or via a quick script. A note can move back and forth.

## Daily Notes System
Daily notes live in `06 - Daily/` with format `YYYYMMDD`. They serve as:
- **Morning check‑in** – priorities, focus
- **Capture area** – quick thoughts, tasks, meetings
- **Evening reflection** – what happened, what I learned

The temporal review cycle:
Daily → Weekly → Monthly → Yearly

Each level reviews the one below it, surfaces highlights, and links forward/backward in time.
- **Weekly notes**: `YYYY-Www` – created manually from `(TEMPLATE) Weekly.md`
- **Monthly notes**: `YYYY-Mmm` – from `(TEMPLATE) Monthly.md`
- **Annual notes**: `YYYY-Y` – from `(TEMPLATE) Yearly.md`
Navigation between daily notes uses inline wikilinks rendered as `← previous | → next` (generated by the daily template).

## Processing the Inbox

1. Open `08 - Nexus/Inbox Dashboard.md` (or use Dataview query)
2. For each note in `00 - Inbox/`:
   - If it's a source: run `Source Capture` again (or move to `01 - Sources/` and tag appropriately)
   - If it's a fleeting thought: decide to discard, expand into a literature/permanent note, or move to an existing project
   - If it's a task: move to `05 - Projects/` or the relevant daily note
1. After processing, move the note out of `00 - Inbox/` (or delete it).

## Workflows by Source Type

### Book Workflow

Book → Zotero → Source Note → Literature Note → Permanent Notes → MOCs

**Tools:** Zotero + Better BibTeX + Zotero Integration plugin.
**Highlight convention:**

| Colour | Meaning |
|--------|---------|
| Yellow | Important to author |
| Green | Important to me |
| Pink | Disagreement |
| Purple | Investigate later |
| Blue | Added later |


---

### Research Paper Workflow

Research Rabbit → Connected Papers → Zotero → Annotations → Literature Notes → Permanent Notes


**Purpose:** Transform academic research into reusable knowledge.

---

### Article Workflow

Article → Raindrop → Source Note → Literature Note → Permanent Note

Articles are processed lighter than books.

---

### YouTube Workflow

Video → Source Capture → Timestamp Notes → Literature Note → Permanent Notes


**Tip:** Use timestamp links: `[1:42](youtube-link)`

---

### Podcast Workflow

Podcast → Snipd / Airr → Timestamp Notes → Literature Note → Permanent Notes


Extract ideas, not transcripts.

---

### Thought Workflow
  
Not every thought deserves development. The growth system lets ideas mature naturally.

---

### Lecture Workflow

Lecture → Source Capture → Course → Unit → Literature Note → Permanent Notes


Lecture notes automatically link to Course MOC, Unit MOC, and Lecturer – creating navigable academic structures.

## Temporal Workflows

### Daily Workflow

- **Morning:** Review tasks, review due notes.
- **Throughout the day:** Capture ideas, create sources, process inbox.
- **Evening:** Reflect, link notes, schedule reviews.

### Weekly Review

Review daily notes, open projects, and new permanent notes.  
Ask: *What did I learn? What deserves further development?*

### Monthly Review

Review weekly notes, active MOCs, and growth statistics.  
Identify emerging themes and neglected areas.

---

## Evergreen Development Workflow

A note becomes evergreen when it is:

- Atomic
- Understandable alone
- Linked to other ideas
- Supported by evidence
- Useful for future thinking

Evergreen is not a finish line – it is the current best version of an idea.

---

## Review Workflow

Review dates are stored in frontmatter: `review: YYYY-MM-DD`.  
Sources receive a review date automatically (14 days after creation).  
Reviews surface notes that deserve additional thinking.

---

## Discovery Workflow

Ideas are rediscovered through:

- Dataview dashboards
- Random Note plugin
- Graph traversal
- MOCs
- Search

Discovery is as important as capture. A note never revisited has little value.

---

## Knowledge Compounding

More Notes ≠ More Knowledge

Knowledge compounds when ideas become **connected**.  
ShadowVault is designed to maximise those connections over time.
